
# FCast Sender

A `sender` is a device or software application that discovers and communicates with a `receiver`. The sender uses the FCast protocol to send media content to the receiver, such as a TV, media top box, or desktop.
The sender initiates the media streaming by connecting to the receiver, launching the media, and then the receiver begins playing the media. Once the media is launched, the sender can control the playback, allowing operations like pause, resume, seek, and volume adjustment.

Applications that supports the FCast protocol:

* From FUTO
    * [**Grayjay**](https://grayjay.app/) is a player that exposes multiple video websites as sources in the app.
    * [**Terminal**](/sender/terminal/) sender allows you use the FCast protocol over the command line interface.
    * [**Desktop**](/sender/desktop) sender allows you to mirror your screen and cast files from your file system.
* From third parties
    * [**CloudStream**](https://github.com/recloudstream/cloudstream) Android app for streaming and downloading media.
    * [**CastLab**](https://codeberg.org/tom79/CastLab) is an app that allows to cast gallery media through DLNA protocol.
    * [**ReFra**](https://github.com/IacobIonut01/ReFra) Media Gallery app for Android made with Jetpack Compose.

If you want to have your own app on this page please open an [issue](https://github.com/futo-org/fcast/issues) or a [pull request](https://github.com/futo-org/fcast/pulls) and we will add it.
